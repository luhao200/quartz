## 实际编程中的注意事项

考虑[extern](存储类.md#extern%20存储类)的[数据类型](数据类型.md)，尽可能节省资源
注释要从业务层面写，而不是写程序本身的意思
多行注释
    \#if 0
    \#endif
    优于/\*\*/
尽量不用break和continue，当循环内内容很多时，容易打破代码整体性

拿到方案先思考，然后再开始
    比如，要求矢量在0-180度之间可以转换为虚部大于0
根据具体项目来选择用==时间换空间还是用空间换时间==

尽量用范围更小的意义更明确的方法
    比如定义一个判断变量时，bool类型比int类型更明确

对比

|     | for | 递归  |
| --- | --- | --- |
| 可读  |     | 好   |
| 性能  | 好   |     |

|        |                                                                                 |     |
| ------ | ------------------------------------------------------------------------------- | --- |
|        | 宏函数                                                                             | 函数  |
| 执行效率   | 快                                                                               |     |
| 空间占用   | 大                                                                               |     |
| 参数类型   | 无关                                                                              |     |
